# LexAnchor (锚点) · 合同版 v0.1 - 产品需求与设计文档 (PRD)

## 1. 产品概述
**产品名称:** 锚点（LexAnchor）· 合同版  
**一句话定义:** 锚点（合同版）是一个针对 SaaS / Vendor 合同的红旗条款初筛工具，将风险条款精准锚定到原文位置，并输出结构化审查结果。

## 2. 目标用户
本项目第一版精准切入具体执行层，避免泛泛定位为“律师”：
- **企业法务 (In-house Counsel)**
- **采购法务 / 商务法务**
- **初级律师 (Junior Associate)**

*(注：合伙人通常不直接使用工具逐条“扫雷”，而是聚焦关键商业问题与最终风险决策，因此不作为首批目标用户。)*

## 3. 核心使用场景
本产品售卖的核心是“场景”而非单纯的“工具”。这三个场景足够撑起一个基础产品：

- **场景 1：合同进件第一步（法务前置初筛）**
  > “我先丢进锚点扫一遍，看看有哪些大坑，再决定怎么分配精力深入审查。”

- **场景 2：业务方防微杜渐（业务自查）**
  > “提交法务审批前，我先自己跑一遍，避免因为低级的红旗条款被法务直接打回来。”

- **场景 3：新人审查辅助（兜底防漏）**
  > “刚入职或接手新类型合同，我怕漏掉致命的红旗条款，用锚点作为安全兜底。”

## 4. 第一版（v0.1）核心锚点设计
为了保证极高的准确率、避免贪多嚼不烂，v0.1 收敛至 **9 个绝对核心锚点**，按处理逻辑和重要性分为三个层级：

### 🧨 Level 1: 核心价值（必须做，红旗条款核心）
*此类条款一旦出现且不对等，将带来巨大商业或法律风险。*
1. **无限责任 (Unlimited Liability)**
2. **单方免责 / 单方赔偿 (Indemnity Imbalance)**
3. **单方修改权 (Unilateral Change)**
4. **自动续约 (Auto Renewal)**

### ⚠️ Level 2: 结构性缺失（非常重要，合同完整性检查）
*此类问题通常是“找不到”带来的风险，考验系统对全文结构的理解。*
5. **缺少终止条款 (Missing Termination Clause)**
6. **缺少数据删除 / 返还条款 (Missing Data Deletion/Return)**
7. **缺少适用法律 / 管辖条款 (Missing Governing Law/Jurisdiction)**

### 🧠 Level 3: 半结构化/复杂判定（需要 LLM 深度参与）
*此类条款涉及具体商业条件的博弈，需大模型进行语义和程度的判断。*
8. **知识产权归属异常 (IP Ownership Anomalies)**
9. **审计权过宽 (Overbroad Audit Rights)**

## 5. 输入与输出规格 (I/O Specifications)
**输入:**
- 一份合同文件（支持格式：PDF / Word / 纯文本）

**输出:**
- **风险条款清单:** 包含具体的风险类型，并**精准带有原文引用（锚定到具体位置/页码）**。
- **缺失条款报告:** 明确指出 Level 2 中缺失的结构性条款。
- **风险等级评估:** 对整体合同或单一条款给出风险评级。
- **人工复核建议:** 针对模糊地带输出“是否需要人工复核 (Human-in-the-loop)”的布尔值及简要理由。
